## Switch points

Model switches happen only at gate boundaries and retirement points.
They never happen mid-session. The gate signals that trigger a model
switch are defined in `knowledge/dependency_graph.md`.

| Gate signal | Switch direction | Reason |
|---|---|---|
| `[C1_INTERFACES_PUBLISHED]` | → Claude Sonnet for C4 | Physics traits require Tier A |
| `[C4_INTERFACES_PUBLISHED]` | → Gemini 3.1 Pro for C5 scaffolding | Sub-coordinator boilerplate |
| Any `[NEEDS_REVIEW: claude]` batch | → Claude Sonnet for C7 review | Policy requirement |
| `severity: critical` or `severity: arch-break` bug | → Claude Sonnet | Policy requirement |
| `[CX_COMPLETE]` for a Gemini-owned coordinator | → Claude Sonnet if next role is Tier A | Per table above |

## How to use this table in a handoff prompt

Every handoff prompt must include a `Model:` field:

```
Model: <model name from table above>
```

The agent writing the handoff prompt looks up its successor role in the
table and copies the model name verbatim. If the successor role spans
two models (e.g. C3), use the model appropriate for the next immediate
task, not the full role lifecycle.
